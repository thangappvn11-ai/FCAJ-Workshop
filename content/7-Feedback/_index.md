The First Cloud AI Journey internship gave me a valuable opportunity to learn AWS in a structured way, practice with real cloud services, and improve my ability to document technical work. Besides technical knowledge, the program also helped me understand how to approach cloud projects more professionally: define requirements, select suitable services, estimate cost, test carefully, and clean up resources after practice.

### Overall Evaluation

**1. Working Environment**  
The learning environment was open, friendly, and focused on practical improvement. I felt comfortable asking questions and reviewing technical issues with mentors and program members. This helped me stay motivated throughout the internship.

**2. Support from Mentor / Team Admin**  
Mentors and admins provided clear guidance, useful learning resources, and timely support when I faced difficulties. The feedback I received helped me improve both technical understanding and report quality.

**3. Relevance of Work to Academic Major**  
The internship was closely related to my Information Technology major. Topics such as cloud computing, networking, security, database, system design, and technical documentation matched the knowledge I learned at university while expanding it with practical AWS experience.

**4. Learning & Skill Development Opportunities**  
I learned how to work with AWS services such as EC2, S3, RDS, CloudFront, WAF, SQS, Transcribe, Cognito, SES, CloudWatch, and VPC Endpoints. I also improved technical writing through blog translation, proposal writing, and workshop documentation.

**5. Company Culture & Team Spirit**  
The program encouraged a learning mindset, responsibility, and knowledge sharing. Even when the topics were challenging, the support from mentors and peers made the learning process more positive and effective.

**6. Internship Policies / Benefits**  
The program structure was clear and gave interns enough room to learn, practice, document, and improve. The combination of weekly work, project proposal, blogs, events, and workshop activities created a complete learning path.

### Personal Reflection

The most satisfying part of the internship was being able to connect AWS theory with practical tasks, especially through the VDCMS architecture proposal and the S3 VPC Endpoint workshop. These activities helped me understand how cloud services are selected and combined to solve real problems.

For future interns, I think the program could provide more short review sessions for complex topics such as networking, IAM, and cost optimization. These topics are important but can be difficult for beginners.

I would recommend this program to friends who are interested in cloud computing because it provides a practical learning path, useful technical exposure, and opportunities to improve documentation and presentation skills.

### Suggestions

* Add more short technical review sessions after major AWS topics.
* Provide sample checkpoints for project progress so interns can self-review earlier.
* Continue encouraging interns to write, translate, and share technical content because it helps strengthen understanding.
